## 追加確認 ❺：時間重複バリデーションの現状仕様について

先日いただいたご質問「キャンセルした日時と同じ日時の別求人にはそもそも応募ができなくなっている認識ですがいかがでしょうか」につきまして、社内で再調査を行いました結果、当初お伝えした内容を以下のとおり訂正させていただきます。

### 現状（再調査結果・正）

**❶ 確定済の応募（マッチ済・勤務中）と時間が重なる別求人への応募は、画面側でブロックしています。**
求人詳細・求人カードで応募ボタンが「時間重複」のラベル付きで無効化されます。

**❷ ただし、キャンセル済の応募は対象外です。**
キャンセルされた応募は「確定済」の枠から外れるため、その時間帯と重なる別求人には応募可能な状態になります。

**❸ 審査待ち（APPLIED）の応募も対象外です。**
審査マッチング求人で応募中の段階では時間重複扱いになっていません。

### ご確認内容
今回の初心者応募上限とあわせて、以下のいずれかをご選択いただきたく存じます。

- **案①（現状維持を推奨）**：現状の時間重複ブロックは触らない。初心者応募上限（同時2件）のみ導入。キャンセル後の同時間帯への再応募は制限しない。
- **案②**：時間重複ブロックの対象を拡大し、キャンセル済・審査待ちも含めて「同じ日時の別求人」への応募を抑止する。
- **案③**：案②に加えて、サーバー側にも時間重複チェックを追加する（堅牢性向上）。

### 補足
ご懸念の「初心者の繰り返しキャンセル」は、今回の同時2件上限により**新規応募の天井が抑えられる**ため、案①でも一定の抑止効果は見込めます。
案②・案③は本要望の必須要件ではなく、別フェーズでの対応も可能です。

## 追加確認 ❻：「N回以上勤務」条件付き求人と初心者応募上限の関係

タスタスには「N回以上勤務（2〜5回）」を応募条件とする求人があり、応募時は条件回数分の日程をまとめて選択する必要があります。

例：「週3回以上勤務」が条件の求人 → ワーカーは3日分を選んで応募

このとき、勤務実績のないワーカーが応募しようとすると、「3日まとめ応募」と「同時2件上限」が衝突します。以下のいずれかを選択いただきたく存じます。

### 選択肢

- **案A（推奨）**：初心者上限のカウント単位を「求人単位」とし、同一求人の複数日応募は **1件としてカウント**。
  - 例：週3回以上勤務の求人に3日応募 → カウントは1件 → あと1件まで別求人に応募可能
- **案B**：「N回以上勤務」条件付き求人は、初心者上限の対象外とする（オファー・限定求人と同じ扱い）。
- **案C**：勤務実績のないワーカーは「N回以上勤務」条件付き求人に応募不可とする。

### 補足
案Aがカイテクの考え方に近く（カイテクは同時応募上限が「件数」単位＝1求人1カウント）、運用上の自然さでも有力です。
案Cはご要望のキャンセル抑止に最も強い効果がありますが、ワーカーの応募機会を大きく狭めます。
