# 日払い機能 UX/フロー設計レビュー

**レビュー日**: 2026-05-27
**レビュアー**: Codex (UX Architect)
**基準**: PayPay/LINE Pay/メルペイ系の親しみやすさ
**対象ユーザー**: 看護師・介護士（30〜60代女性中心、スマホ利用）

---

## 1. Bottom Line

全体として、仕様は成立していますが、このまま画面化すると「報酬受け取り系の申請画面」になりやすく、PayPayのような直感性にはまだ足りません。最重要は、ワーカーに「今いくら受け取れるか」「何をすれば増えるか」「失敗したらどう直すか」を1画面で見せることです。

## 2. 致命的なUX問題（P0）

- **P0-A**: レビュー提出が前払い解放条件なのに、導線が別機能に見える。「レビューしないとお金が受け取れない」と理解しづらい。
- **P0-B**: 口座登録/変更がプロフィール編集に埋もれると重すぎる。銀行名、支店、口座番号、本人認証が同居すると、夜勤明けユーザーには離脱要因。
- **P0-C**: "前払い率・プール・控除・組戻し"の業務用語がワーカー向けに不向き。「今すぐ受け取れる金額」「支払日に入る金額」に変換すべき。
- **P0-D**: エラー時の安心導線が未定。「振込エラー」だけでは不安が強い。原因、直す場所、再申請可否を1画面で出す必要。

## 3. ワーカー側フローの改善案

| フロー | 現状想定 | 目標 | 改善案 |
|---|---|---|---|
| **A1 残高確認** | 2〜3ステップ | 0〜1ステップ | 5タブを「探す/仕事/お金/メッセージ/マイページ」にし、`お金`タブで大きく「¥12,480 今すぐ受け取れます」 |
| **A2 引き出し申請** | 5〜6ステップ | 3ステップ | `お金` → 金額入力 → 確認ボトムシート → 完了。手数料と振込予定額を同じ画面に |
| **A3 口座登録/変更** | 6〜8ステップ | 登録3、変更4 | 登録: 口座入力 → 確認 → 完了。変更: 口座入力 → SMS/メール認証 → 確認 → 完了。プロフィール編集とは分離 |
| **A4 履歴閲覧** | 2〜4ステップ | 1〜2ステップ | `お金`画面内に「履歴」タブ。`受け取り済み/申請中/支払日に入る`で色分け |
| **A5 出金エラー** | 5ステップ以上 | 3ステップ | 通知 → エラー詳細 → 口座修正/再申請。エラーコードは隠し、原因を日本語化 |
| **A6 レビュー提出** | 4ステップ | 2ステップ | `レビューすると ¥8,640 受け取れます`カード → 30秒レビュー → 残高反映。自由記述は任意またはチップ選択中心 |

### 共通マイクロインタラクション

- 残高反映時に数字が軽くカウントアップ
- `レビュー → 残高に追加 → 受け取り`のステップ表示
- 申請後は `申請受付 → 銀行確認中 → 振込完了`の進行表示
- 完了時コピー: 「受け取り申請できました」+ 軽いチェック演出
- 失敗時は赤一色ではなく、`口座を直す`の主ボタンを最上部に固定

### アクセシビリティ

- 本文16px以上、金額36〜44px、ボタン高さ56px推奨
- タップ領域は最低48px
- 金額入力は数字キーボード、全額ボタン、スライダー併用
- 赤だけで状態を伝えず、アイコンと文言を併用
- 高齢寄りユーザー向けに、薄いグレー文字は避ける

## 4. 管理者側フローの改善案

| フロー | 目標ステップ | 改善案 |
|---|---|---|
| **B1 個人別率/上限設定** | 3 | ワーカー詳細 → 前払い設定 → `通常90% / 控えめ70% / 停止`プリセット選択。理由メモ必須 |
| **B2 緊急一括停止** | 2 | 管理ダッシュボード最上部に赤い固定スイッチ。押下後、確認モーダルで停止。全画面に停止中バナー |
| **B3 エラーモニタリング** | 3 | エラー一覧 → 対象確認 → 通知/再処理。`口座不一致/銀行停止時間/残高不足`で自動分類 |
| **B4 履歴閲覧** | 2 | 検索・フィルタ付きテーブル。CSV出力あり |

管理者UIはPayPay風に寄せすぎず、業務ダッシュボードとして密度高めでよい。ただし「緊急停止」と「出金エラー」は別メニューに埋めず、常に見える場所に。

## 5. 施設側非表示方針の評価

個別の前払い実行有無は、施設側には**非表示でよい**。施設に見せると「施設が承認・管理するもの」と誤解され、問い合わせや運用負荷が増える。

ただし、施設側には最低限の説明が必要:
> 勤務完了後の報酬処理はタスタス側で行います。施設側で前払い申請を確認・承認する必要はありません。

レビュー提出が報酬処理と関係するなら、施設マニュアルやFAQに「レビュー依頼が届く場合があります」程度は入れると問い合わせを減らせる。

## 6. PayPay系から採用すべきパターン

1. **大きな残高表示**: `¥12,480`を最上部に太字表示。ラベルは「今すぐ受け取れる金額」
2. **主要アクション1つ**: 残高カード内の主ボタンは `受け取る` のみ。補助は `履歴`、`口座`
3. **フラットな5タブ**: `探す/仕事/お金/メッセージ/マイページ`。日払いはマイページ配下ではなく独立タブ
4. **履歴タイムライン**: `チャージされました/申請中/振込完了/確認が必要`をアイコン付きで表示
5. **ボトムシート確認**: 申請前の確認は画面遷移ではなく、下から出る確認シートで軽く済ませる
6. **友好的な言葉**: `申請`より`受け取る`、`前払い可能額`より`今すぐ受け取れる金額`

## 7. コピーライティング提案

| Before | After |
|---|---|
| 前払い申請 | 受け取る |
| 前払い可能額 | 今すぐ受け取れる金額 |
| レビュー未提出のため対象外 | レビューすると ¥8,640 受け取れます |
| 振込エラー | 口座を確認できませんでした |
| 口座修正を促す | 受取口座を直してください |
| 手数料100〜200円本人負担 | 手数料 ¥143 を引いて振り込みます |
| 1円単位で引き出し可能 | 1円から受け取れます |
| 月末締め | 5/31 23:59まで受け取りできます |
| 前払い率7割 | 一部は支払日に入ります |
| レビューを投稿してください | 30秒レビューで受け取りできます |
| 投稿中... | 送信しています |
| その口座では再引き出し不可 | この口座では受け取れません。別の口座を登録してください |
