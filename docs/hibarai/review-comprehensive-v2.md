## 1. 訂正サマリ

前回レビューは `docs/hibarai/report-2026-05-07.html` を読まずに作成されたため、確定済み事項を「未確定」と誤判定していた。以下は **report を踏まえた訂正と、業務委託前提の現方針への補正**：

| 前回の誤判定 | 訂正（report 章・行を引用） |
|---|---|
| 報酬支払日が未確定/衝突 | report 2章は固定日支払い前提だったが、現方針では**案件ごとの既存報酬支払日を維持**。月末23:59締め、翌1日精算、残額は各案件の既存支払日に乗せる |
| 前払い率/算定方針が未確定 | **基本9割、労働者性が否認できない場合の留意等は7割、算出要素は確定済み**。report 1.2 / 3章 |
| ポイント残高化が未確定 | **確定報酬は自動的にポイント残高加算（1pt=1円ウォレット型）で確定済み**。report 1.2 / 2章 |
| 月末締めサイクルが未確定 | **月末23:59締切、翌1日精算、既存支払日に残額支払で整理**。report 2章 |

既存の案件別支払日設定がある場合は、日払い機能側で統一せず、**既存支払日を参照する実装対象**として扱う。

## 2. 新しい総合評価

**総合成熟度 68%**。事業仕様の中核「確定報酬 → ポイント残高 → ワーカー申請 → GMO即時振込 → 既存支払日での精算」が report ベースで整理済み。一方で、Prisma本設計・Server Actions・GMO連携に入るには report 5.1〜5.3 の未解決項目が残る。

実装面はダミーデータ中心。日払い台帳、ポイント残高、出金、GMO取引、月次精算を表す Prisma model は現行 `schema.prisma` にない。既存接続点は `Attendance` の計算済み報酬フィールド、`Review`, `BankAccount`。

## 5. 新たな指摘

### 5.1 screen-spec-draft.md が report を参照していない
参照は議事録メモと過去レビュー中心。未確定リストに「前払い可能額の算定母数」が残っているが、**report 3章で確定済み**のため、「仕様未確定」→「実装データ契約の定義」に格下げすべき。

### 5.2 cycle.html のGMO方式記述が report と不整合
cycle は「GMOあおぞらAPI **または** 即給byGMO」としているが、**report 1.1 は GMOあおぞらAPI のみ**を明示。cycle.html の修正が必要。

### 5.3 ダミーデータの支払日の説明が不足
`lib/dummy-data/hibarai.ts` は `6月25日（水）`。**案件ごとの既存報酬支払日**である旨を明示する必要。

## 8. 次のMTGで決めるべきこと

report 5.x ベースで6項目に絞り込み：

1. **GMO**: 名義照合API、仕向Webhook、組戻しAPI、手数料改定、レート制限、エラーコード表

6. **実装データ契約**: `確定額(交通費込み)` を Prisma 上でどの台帳・どの時点のスナップショットから作るか
