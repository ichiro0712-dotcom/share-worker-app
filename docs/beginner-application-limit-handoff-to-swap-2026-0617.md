## 1. 仕様確定のご報告

当方タスク「勤務実績なしワーカーの応募求人数制限」につきまして、クライアントの追加確認が完了し、全項目仕様確定いたしました。

### 確定事項（全項目）
- 「勤務実績なし」= `Application.worker_review_status === 'COMPLETED'` が 0 件のワーカー
- 同時応募上限: **2 件**
- カウント対象: APPLIED / SCHEDULED / WORKING / COMPLETED_PENDING（CANCELLED は除外）
- 上限対象外: オファー、限定求人 (LIMITED_*)
- 解除: 1件完了で恒久解除
- **weekly_frequency 求人**: **初心者は応募不可**（案B採択）
- 時間重複ブロック: 現状維持（将来 F1 で拡張予定）
- 既存ワーカー: 強制キャンセルなし・自然減衰

## 2. 先方の保留 H3 への結論

### H3「振替応募を応募数制限のカウント対象にするか」
→ **「考慮不要」で確定可能です。**

理由：
- **案B採択により、初心者は weekly_frequency 求人にそもそも応募できない**
- 振替が発生するのは weekly_frequency 求人のみ
- したがって振替シナリオに到達するワーカーは全員「勤務実績ありの非初心者」＝当方の上限対象外
- 振替時にカウントを増やすか減らすかという議論自体が成立しない

→ **先方は振替実装において、初心者上限を意識する必要がありません。**
